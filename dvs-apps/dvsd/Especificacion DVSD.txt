DVSD: Es un servidor distribuido que permite la gestion integral del cluster DVSD

El sistema esta constituido por 3 tipos de procesos:
	1- MONITOR (WEBMIN) Permite ejecutar comandos en todos o en algunos nodos del DVS.
	2- PROXY (usa spread) : Es el responsable de realizar el multicast 
		de las peticiones recibidas desde el MONITOR hacia los diferentes nodos 
		y de recibir los resultados.
		El Proxy se comunica con NAMED PIPES con el MONITOR y por SPREAD con los AGENTS.
		Para esto utiliza 2 hilos: El hilo principal que recibe los comandos desde 
		el MONITOR y hace el multicast por spread, y el hilo receptor de spread que 
		recibe los resultados y los devuelve al monitor por NAMED PIPE.
	3- AGENT: se ejecutan en los nodos a controlar (usa spread). Recibe por spread el 
		requerimiento y devuelve por spread el resultado.
		
Las peticiones (CMDIN) y las respuestas (CMDOUT) son atomicas en spread, 
es decir solo hace falta un unico mensaje para contenerlos.


Cada peticion RQST_XXXXXX tiene asociado:
	- nodo peticionante
	- nodo destinatario (puede ser DVS_ALLNODES)
	- Numero de secuencia del peticionante
	- tipo de requerimiento
	- parámetros del requerimiento
	
Cada peticion RQST_RPLY tiene asociado:
	- nodo respondiente
	- nodo peticionante
	- Numero de secuencia del peticionante
	- tipo de requerimiento
	- codigo de error (rcode)
	- número de respuestas (rply_count) o longitud de respuesta (rply_len)
	- array de respuestas

RESTA RESOLVER:
	1) Cuando se hace un pedido simple (por ej. GETDCINFO) pero a DVS_ALLNODES
		Como se responde? 
			a) NODOi -> DVSD -> DVSCMD (todas respuestas individuales)
	>> ESTA	b) NODOi -> DVSD quien acumula respuestas y las envia luego al DVSCMD
			
	2) Cuando se hace un pedido de multiples lineas (por ej. GETPROCINFO) pero a DVS_ALLNODES
		Como se responde? 
			NO ES VALIDO - Solo se pueden enviar pedidos de multiples lineas a nodos individuales

RQST_DVSINIT 
============
Requerimiento:
	- dvs_usr_t;
Respuesta:
	- rcode = nodeid | ERROR_CODE
	- rply_count = 0

RQST_GETDVSINFO 
===============
Requerimiento:
Respuesta:
	- rcode = ERROR_CODE
	- rply_count = 1
	- dvs_usr_t;
	
RQST_DVSEND  
===============
Requerimiento:
Respuesta:
	- rcode = ERROR_CODE
	- rply_count = 0
 
RQST_DCINIT
===========
Requerimiento:
	- DCID
	- DC_usr_t
Respuesta:
	- rcode = DCID | ERROR_CODE	
	- rply_count = 0

RQST_GETDCINFO
===============
Requerimiento:
	- DCID
Respuesta:
	- rcode = ERROR_CODE	
	- rply_count = 1
	- DC_usr_t
 
RQST_DCEND
===========
Requerimiento:
	- DCID
Respuesta:
	- rcode = ERROR_CODE
	- rply_count = 0
	
RQST_ADDNODE
===========
Requerimiento:
	- DCID
	- nodeid
Respuesta:
	- rcode = ERROR_CODE
	- rply_count = 0

RQST_GETNODEINFO 
===========
Requerimiento:
	- nodeid // en el nodo destino existen varias definiciones de nodos
Respuesta:
	- rcode = ERROR_CODE
	- rply_count = 1
	- node_usr_t

RQST_DELNODE 
===========
Requerimiento:
	- DCID
	- nodeid
Respuesta:
	- rply_count = 0
	- rcode = ERROR_CODE
	
RQST_GETPROXYINFO 
=================
Requerimiento:
	- PXID
Respuesta:
	- rcode = ERROR_CODE	
	- rply_count = 2 // son 2  proxies
	- proc_usr_t[2]

RQST_GETPROXYSTS 
=================
Requerimiento:
	- PXID
Respuesta:
	- rcode = ERROR_CODE	
	- rply_count = 1
	- proxies_usr_t
	
TRATAMIENTO DE MENSAJES SPREAD
==============================
JOIN:   Todos incorporan el nodo al DVS
			Verifican en la configuración que script van a ejecutar por JOIN
		
DISCONNECT: Todos remueven el nodo del DVS
			Verifican en la configuracion que script van a ejecutar por DISCONNECT
			
PARTITION: Todos refrescan la configuracion del DVS
			Verifican en la configuracion que script van a ejecutar por PARTITION

MERGE:
			Verifican en la configuracion que script van a ejecutar por MERGE

APPLICATION MESSAGE: 
	Da tratamiento individual
